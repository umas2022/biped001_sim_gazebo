# 最小 LQR Demo 说明

这份文档对应：

[`examples/lqr_minimal_demo.py`](../examples/lqr_minimal_demo.py)

它现在同时承担两件事：

- 解释 LQR 最基本的概念
- 逐段解释这个最小示例代码

这样你只需要看这一份文档，就能把最小例子看明白，不需要再在多份文档之间来回跳。

## 1. 这个 demo 的定位

它不是正式控制器，也不是高保真物理模型。

它只是一个教学骨架，刻意只保留四步：

1. 定义离散状态空间模型
2. 定义 `Q` 和 `R`
3. 求出 LQR 增益 `K`
4. 用 `u = -Kx` 跑一个小仿真

它不依赖：

- ROS 2
- Gazebo
- 图形界面

所以更适合拿来理解控制主线。

## 2. LQR 最短解释

LQR 是一种状态反馈控制方法。

它依赖的不是神经网络模型，而是一个数学模型：

`x[k+1] = A x[k] + B u[k]`

这里：

- `x`
  是状态向量
- `u`
  是控制输入
- `A`
  描述系统自己会怎么演化
- `B`
  描述控制输入会怎样影响系统

LQR 最终求出一个反馈增益：

`u = -Kx`

也就是根据完整状态 `x` 来决定控制输入，而不是只盯着单个误差项。

## 3. 它和 PID 的区别

PID 更像是围绕一个误差做反馈：

`u = Kp * e + Ki * ∫e dt + Kd * de/dt`

LQR 更像是：

- 先写出整个系统的状态模型
- 再用一个统一的代价函数权衡状态误差和控制输出
- 最后得到状态反馈增益 `K`

所以对两轮平衡车这种多状态耦合系统，LQR 往往比单环 PID 更自然。

## 4. 这个最小例子的状态是什么

示例里只用两个状态：

`x = [theta, theta_dot]`

也就是：

- `theta`
  倒立摆相对竖直方向的倾角
- `theta_dot`
  倾角变化速度

这比正式项目简单很多。

正式项目里使用的是四状态：

- `wheel_pos_error`
- `wheel_vel`
- `pitch_error`
- `pitch_rate`

所以这个 demo 可以看成：

“先学最简单的两状态倒立摆 LQR，再回去看工程版四状态控制器。”

## 5. 代码结构非常简单

文件里只有两个主要函数：

1. `solve_lqr_gain(a, b, q, r)`
   根据 `A/B/Q/R` 求出 `K`
2. `main()`
   定义模型、设置初值、跑仿真并打印结果

主线可以压缩成：

```python
a, b = ...
q, r = ...
k = solve_lqr_gain(a, b, q, r)

x = initial_state
for each step:
    u = -Kx
    x = A x + B u
```

## 6. `solve_lqr_gain(...)` 在做什么

这个函数对应的离散系统是：

`x[k+1] = A x[k] + B u[k]`

LQR 想最小化长期总代价：

`sum(x^T Q x + u^T R u)`

含义很直接：

- 状态偏差太大，要罚
- 控制输出太大，也要罚

函数内部先用 `solve_discrete_are(...)` 求 Riccati 方程的解 `P`，再由 `P` 计算最优反馈增益 `K`。

初学阶段不需要先背推导，先记住这条链路就够了：

`A/B` 定义系统，`Q/R` 定义偏好，LQR 据此求出 `K`。

## 7. `main()` 里每一步的控制意义

### `dt = 0.02`

表示离散仿真每 20 ms 更新一次，也就是 50 Hz。

### `A`

```python
a = np.array([
    [1.0, dt],
    [9.0 * dt, 1.0],
])
```

它表达的是：

- 当前角度会受到当前角速度影响
- 当前倾角会继续推动系统往“更倒”的方向发展

这个写法是简化教学模型，不是完整实体推导。

### `B`

```python
b = np.array([
    [0.0],
    [1.0 * dt],
])
```

表示控制输入 `u` 先影响角速度，再通过角速度影响角度。

### `Q`

```python
q = np.diag([40.0, 4.0])
```

表示更重视把 `theta` 压小，其次才是 `theta_dot`。

### `R`

```python
r = np.array([[0.5]])
```

表示不希望控制输出过于激进。

粗略直觉是：

- `Q` 大一些，控制通常更积极
- `R` 大一些，控制通常更保守

## 8. `u = -Kx` 到底怎么落地

仿真循环里最核心的一行是：

```python
u = -(k @ x).item()
```

它表示：

- 当前状态偏了多少
- 当前状态变化有多快
- 这些量一起经过增益矩阵 `K`
- 生成一个负反馈控制量 `u`

前面的负号就是“往回拉”的意思。

如果状态偏到正方向，控制要往负方向拉；反过来也一样。

## 9. 限幅和状态更新

后面两步是：

```python
u = max(-5.0, min(5.0, u))
x = a @ x + b.flatten() * u
```

第一行是工程上常见的输出限幅。

第二行就是把离散模型真正跑起来：

- `A x`
  系统自己的演化
- `B u`
  控制输入带来的变化

## 10. 终端输出应该怎么看

脚本会打印：

- `A`
- `B`
- `Q`
- `R`
- `K`
- 随时间变化的 `theta`、`theta_dot`、`u`

看结果时重点观察三件事：

1. `theta` 是否持续接近 `0`
2. `theta_dot` 是否也逐渐接近 `0`
3. `u` 是否在合理范围内逐步减小

如果这三件事同时发生，说明闭环在收敛。

## 11. 这个例子的简化边界

这个 demo 适合回答：

- LQR 的最小闭环长什么样
- 程序里怎么从 `A/B/Q/R` 走到 `u = -Kx`

它不适合回答：

- 真正机器人为什么一定这样建模
- 为什么这个参数能直接上真机
- 为什么加入饱和后仍然严格最优

因为它本来就只是教学版最小例子。

## 12. 从最小 demo 过渡到正式项目

回到正式控制器时，你可以这样建立映射：

- 这里的 `x`
  对应工程版更完整的状态向量
- 这里的 `u`
  对应工程版轮子控制输出
- 这里的 `A/B`
  对应工程版的线性近似模型
- 这里的 `Q/R`
  对应工程版调参偏好
- 这里的离散循环
  对应工程版每个控制周期的读状态、算输出、发命令

骨架没有变，只是工程版状态更多、保护更多、链路更多。
